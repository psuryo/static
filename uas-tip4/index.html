Ringkasan Tugas

Buat sebuah website aplikasi HTML fiktif yang terdiri dari minimal 4 halaman HTML. Website harus menerapkan seluruh elemen yang dipelajari di Modul 01–09, termasuk penggunaan localStorage untuk menyimpan preferensi atau data sederhana pengguna. 01
Background
Latar Belakang & Pilihan Tema Aplikasi
Skenario tugas individu

Kamu diminta membangun sebuah website aplikasi HTML secara mandiri — dipilih dari sembilan tema yang tersedia di bawah. Setiap tema memiliki konteks dan kebutuhan konten yang berbeda, namun semua menuntut penerapan elemen HTML yang sama dari Modul 01–09. Website harus berfungsi sebagai aplikasi digital yang lengkap — menampilkan informasi, data terstruktur, form interaktif, multimedia, dan pengalaman yang dipersonalisasi menggunakan localStorage.

Tugas ini dikerjakan secara individu dan dirancang untuk menguji penerapan seluruh topik dari Modul 01–09 dalam satu proyek nyata yang terintegrasi.

03
Requirements
Persyaratan Teknis — Modul 01–09
Elemen wajib per modul
  • Paragraf, gambar, dan tautan — Minimal 3 paragraf deskriptif, semua gambar punya atribut alt, minimal 3 tautan antar halaman dan 1 tautan eksternal.
    Mod 01
  • Tabel data administrasi — Minimal 1 tabel dengan <thead> dan <tbody> berisi data nyata sesuai tema (jam buka, daftar harga, jadwal layanan, atau spesifikasi produk).
    Mod 02
  • Heading hierarki — Setiap halaman punya tepat 1 tag <h1>, dan menggunakan <h2>/<h3> secara berurutan tanpa melewati level. Gunakan <div> dan <span> sebagai container styling.
    Mod 03
  • List & navigasi — Navigasi utama menggunakan <ul>+<li>+<a>. Minimal 1 unordered list (fitur/menu), 1 ordered list (langkah/prosedur), dan 1 description list (istilah/FAQ).
    Mod 04
  • Semantic HTML5 — Setiap halaman wajib menggunakan: <header>, <nav>, <main>, <article> atau <section>, <aside> (minimal 1 halaman), dan <footer>.
    Mod 05
  • Form kontak — Halaman contact.html berisi form dengan minimal: nama (text, required), email (email, required), subjek (select, required), pesan (textarea, required), dan tombol submit. Semua input punya label yang terhubung dengan benar.
    Mod 06
  • Video atau audio — Minimal 1 elemen multimedia: bisa berupa video promosi bisnis menggunakan <video> dengan atribut controls dan poster, atau embed YouTube menggunakan <iframe> dengan atribut title.
    Mod 07
  • Embed konten eksternal — Minimal 1 <iframe> Google Maps menampilkan lokasi bisnis pada halaman kontak atau beranda. Wajib ada atribut title, loading="lazy", dan style="border:0;".
    Mod 08
  • Meta tags SEO & Open Graph — Setiap halaman wajib punya: <title> unik (max 60 karakter), meta description (max 160 karakter), dan minimal tag OG: og:title, og:description, og:image, og:url. Halaman beranda tambahkan JSON-LD Organization schema.
    Mod 09
Persyaratan localStorage (wajib)

Website harus mengimplementasikan minimal 2 dari 3 fitur localStorage berikut:

👤
Fitur A — Simpan Nama Pengunjung
Form sapaan di beranda: pengunjung mengisi nama, disimpan ke localStorage. Saat kembali ke halaman, nama muncul otomatis: "Selamat datang kembali, [Nama]!"
🌙
Fitur B — Toggle Tema Gelap/Terang
Tombol toggle dark/light mode. Pilihan tema disimpan ke localStorage sehingga tetap aktif saat halaman di-refresh atau pindah halaman.
📋
Fitur C — Simpan Draft Pesan
Form kontak menyimpan isi pesan ke localStorage secara otomatis saat pengguna mengetik. Jika halaman di-refresh sebelum submit, isi form dikembalikan otomatis.
JavaScript — Contoh implementasi localStorage
// ── Fitur A: Simpan & tampilkan nama pengunjung ──
const inputNama = document.getElementById('input-nama');
const sapaan   = document.getElementById('sapaan');

// Cek apakah nama sudah tersimpan
const namaTersimpan = localStorage.getItem('namaPengunjung');
if (namaTersimpan) {
  sapaan.textContent = `Selamat datang kembali, ${namaTersimpan}!`;
}

// Simpan nama saat form dikirim
document.getElementById('form-nama')
  .addEventListener('submit', function(e) {
    e.preventDefault();
    localStorage.setItem('namaPengunjung', inputNama.value);
    sapaan.textContent = `Halo, ${inputNama.value}! 👋`;
  });

// ── Fitur B: Toggle dark/light mode ──
const tema = localStorage.getItem('tema') || 'terang';
if (tema === 'gelap') document.body.classList.add('dark');

document.getElementById('toggle-tema')
  .addEventListener('click', function() {
    document.body.classList.toggle('dark');
    const temaBaru = document.body.classList.contains('dark')
      ? 'gelap' : 'terang';
    localStorage.setItem('tema', temaBaru);
  });

// ── Fitur C: Auto-save draft pesan form kontak ──
const textarea = document.getElementById('pesan');

// Pulihkan draft jika ada
const draft = localStorage.getItem('draftPesan');
if (draft) textarea.value = draft;

// Simpan otomatis tiap pengguna mengetik
textarea.addEventListener('input', function() {
  localStorage.setItem('draftPesan', textarea.value);
});

// Hapus draft setelah form berhasil dikirim
document.getElementById('form-kontak')
  .addEventListener('submit', function() {
    localStorage.removeItem('draftPesan');
  });
💡
Tips localStorage

localStorage hanya menyimpan string. Untuk menyimpan objek atau array, gunakan JSON.stringify() saat menyimpan dan JSON.parse() saat membaca. Data yang disimpan tidak hilang saat browser ditutup — tetap ada sampai dihapus secara eksplisit.

Schedule
Jadwal & Milestone
Minggu 1 — Hari 1–2
Perencanaan & Wireframe
Tentukan tema yang dipilih dan koordinasikan dengan dosen. Buat nama website, identitas bisnis fiktif, dan sketsa wireframe kasar untuk setiap halaman. Buat struktur folder dan file HTML kosong.
Minggu 1 — Hari 3–4
Pembangunan Struktur HTML
Bangun kerangka HTML semua halaman dengan struktur semantik. Pasang navigasi yang terhubung antar halaman. Isi konten teks awal, heading, paragraf, dan list.
Minggu 1 — Hari 5–7
Konten & Komponen Khusus
Lengkapi tabel, gambar dengan alt text, form kontak dengan validasi, embed YouTube dan Google Maps. Pastikan setiap halaman lengkap secara konten.
Minggu 2 — Hari 1–2
localStorage & Meta Tags
Implementasikan minimal 2 fitur localStorage. Lengkapi semua meta tags, Open Graph, dan JSON-LD schema di setiap halaman.
Minggu 2 — Hari 3–4
Review & Perbaikan
Uji semua halaman dan navigasi secara mandiri. Validasi HTML dengan W3C Validator. Cek fungsi localStorage di berbagai browser. Perbaiki semua kesalahan yang ditemukan sebelum dikumpulkan.
