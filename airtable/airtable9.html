02

Automasi: Pekerjaan Berulang jadi OtomatisKonsep Dasar

Automation adalah fitur yang menjalankan tindakan tertentu secara otomatis ketika kondisi tertentu terpenuhi — tanpa admin perlu mengecek dan mengerjakannya manual satu per satu. Contoh sehari-hari di kantor: setiap kali ada pengajuan cuti baru masuk, seharusnya ada notifikasi ke atasan. Tanpa automasi, admin harus mengecek table secara berkala; dengan automasi, notifikasi terkirim sendiri begitu data masuk.

Konsepnya mirip trigger pada database (yang dipelajari sekilas saat membahas SQL/Neon) — perbedaannya, automation Airtable diatur lewat antarmuka visual, bukan ditulis sebagai kode trigger SQL.

03

Anatomi Sebuah AutomationTrigger → Condition → Action

TRIGGER

Record baru ditambahkan

Kapan automation ini mulai berjalan — misal saat ada entri baru di table Cuti Pegawai.

→
CONDITION (opsional)

Jenis Cuti = "Sakit"

Syarat tambahan agar automation hanya berjalan pada kasus tertentu.

→
ACTION

Kirim notifikasi email

Tindakan yang dijalankan otomatis setelah trigger dan condition terpenuhi.

TRIGGER UMUM

When record created

Berjalan setiap kali ada record baru masuk ke table (cocok dipasangkan dengan Form).

TRIGGER UMUM

When record updated

Berjalan saat field tertentu pada record berubah, misal Status berubah jadi "Selesai".

ACTION UMUM

Send email

Mengirim email otomatis ke alamat tertentu, isi bisa memuat data dari record terkait.

ACTION UMUM

Update record

Mengubah nilai field lain secara otomatis, misal mengisi "Tanggal Diproses" dengan tanggal hari ini.

Batasan plan gratis: maksimal 100 kali automation berjalan per bulan, 50 automation per base, dan 25 aksi per automation. Untuk latihan kelas, sebaiknya automation baru diaktifkan setelah struktur data sudah stabil — hindari uji coba berulang-ulang yang menghabiskan kuota bulanan.
04

Langkah PraktikAutomasi Notifikasi Pengajuan Cuti Baru

  1. Buka tab "Automations" di sisi kiri base.Klik "+ Create automation".
  2. Pilih trigger "When record created".Pilih table "Cuti Pegawai" sebagai sumber trigger.
  3. Tambahkan action "Send email".Isi alamat email tujuan (misal email dosen/admin untuk simulasi).
  4. Sisipkan data dinamis ke isi email.Gunakan tombol "Insert field" agar isi email otomatis memuat Nama Pegawai dan Tanggal Cuti dari record yang baru masuk.
  5. Aktifkan automation (toggle "On").Uji dengan mengisi Form Pengajuan Cuti sekali, lalu cek apakah email masuk.
  6. Cek riwayat run di tab "Run history".Airtable mencatat setiap kali automation berjalan, termasuk jika ada error.
