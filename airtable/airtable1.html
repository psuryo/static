02

Apa itu Airtable?Konsep Dasar

Airtable adalah platform pengelolaan data berbasis cloud yang tampilannya menyerupai spreadsheet (seperti Excel), tetapi bekerja seperti basis data relasional di baliknya. Setiap baris adalah sebuah record (mis. satu pegawai, satu surat masuk), dan setiap kolom adalah sebuah field dengan tipe data tertentu (teks, tanggal, lampiran, pilihan, dsb).

Bedanya dengan spreadsheet biasa: antar tabel di Airtable bisa saling terhubung (relasi), persis seperti konsep foreign key yang sudah dipelajari saat membahas SQL dan Neon. Bedanya dengan Access atau SQL murni: Airtable tidak memerlukan penulisan query — struktur relasi, form input, hingga tampilan laporan dibangun lewat antarmuka visual (drag-and-drop).

04

Istilah DasarKartu Kosakata

STRUKTUR

Base

Satu "wadah" proyek — setara satu file database. Berisi kumpulan table yang saling berkaitan.

STRUKTUR

Table

Setara satu tabel di Access/SQL. Contoh: tabel Pegawai, tabel Surat Masuk.

DATA

Record

Satu baris data — satu entri lengkap, misalnya data satu pegawai.

DATA

Field

Satu kolom data dengan tipe tertentu — teks, angka, tanggal, lampiran, dsb.

TAMPILAN

View

Cara menampilkan data yang sama: Grid, Kanban, Calendar, atau Gallery.

RELASI

Linked Record

Penghubung antar table — konsep yang setara foreign key di SQL.

05

Langkah PraktikMembuat Akun & Base Pertama

  1. Buka airtable.com dan daftar akun.Gunakan email kampus jika tersedia agar mudah dikelola dosen.
  2. Pilih "Start from scratch".Hindari template bawaan dulu — tujuan sesi ini adalah memahami struktur dari nol.
  3. Beri nama base: "Latihan Administrasi – [Nama Anda]".Penamaan konsisten memudahkan dosen memeriksa base sekelas.
  4. Buat table pertama bernama "Pegawai".Tambahkan field: Nama, Jabatan, Tanggal Masuk, Status.
  5. Ubah tipe field sesuai datanya.Nama → Single line text, Tanggal Masuk → Date, Status → Single select.
  6. Tambahkan 5 baris data contoh.Gunakan data fiktif, bukan data pribadi sungguhan.
