02

Dari Report Access ke Interface DesignerKonsep Dasar

Di Access, laporan dibuat lewat Report Wizard — hasilnya statis, biasanya untuk dicetak. Di Airtable, fungsi setara ini dijalankan oleh Interface Designer: kanvas terpisah dari Grid tempat Anda menyusun elemen visual (angka ringkasan, grafik, daftar record) yang datanya selalu hidup — begitu data di table berubah, tampilan Interface ikut berubah otomatis, tanpa perlu "generate ulang" laporan seperti di Access.

Interface Designer memakai data yang sudah diolah di sesi-sesi sebelumnya: View yang sudah difilter/sorted/grouped (Sesi 08), dan angka dari Rollup/Formula (Sesi 06-07) — sesi ini adalah tempat semuanya "dipajang" jadi satu tampilan yang mudah dibaca orang non-teknis.

04

Menghubungkan Dashboard ke FormAgar Terasa Seperti Satu Aplikasi

Dashboard yang hanya menampilkan laporan belum terasa seperti "aplikasi" seutuhnya — perlu ada jalan untuk memasukkan data baru tanpa keluar dari tampilan yang sama. Ada dua cara menghubungkan Form ke Dashboard:

CaraBagaimana TampilannyaCocok Untuk
Button → Buka URL FormTombol di Dashboard membuka Form di tab terpisahPihak luar tanpa akun Airtable (pelanggan, pemohon)
Interface Form (halaman baru)Form menjadi satu halaman dalam Interface yang sama dengan DashboardTim internal yang perlu lihat laporan sambil input data

Cara 1 — Tombol menuju Form

  1. Di halaman Dashboard, klik "Add element" → pilih Button.Letakkan di bagian atas Dashboard agar mudah terlihat.
  2. Atur aksi tombol menjadi "Open URL".Tempel tautan Form Pengajuan Cuti dari Sesi 04.
  3. Beri label yang jelas, misal "+ Ajukan Cuti Baru".Untuk proyek restoran: "+ Buat Pesanan Baru".

Cara 2 — Interface Form sebagai halaman baru

  1. Di Interface yang sama, klik "+ New page".Pilih layout "Form".
  2. Pilih table tujuan, misal "Cuti Pegawai".Susun field yang ingin ditampilkan seperti mengatur Form biasa.
  3. Kembali ke halaman Dashboard, tambahkan Button.Atur aksinya menjadi "Go to interface page", lalu pilih halaman Form yang baru dibuat.
  4. Uji coba: klik tombol dari Dashboard, isi Form, lalu kembali.Data baru akan langsung tercermin di elemen Dashboard tanpa perlu refresh manual.
Kaitan dengan proyek restoran: gunakan Interface Form untuk peran internal seperti Kasir (input pesanan sambil melihat status dapur di halaman yang sama), dan Button ke Form/share link terpisah untuk pelanggan luar yang memesan secara mandiri.
05

Langkah PraktikMembangun Dashboard Cuti Pegawai

  1. Klik tab "Interfaces" di sisi kiri base, buat Interface baru.Pilih layout kosong ("Start from scratch") agar bebas menyusun sendiri.
  2. Tambahkan Number Block untuk "Total Pengajuan".Hubungkan ke Rollup COUNT dari Sesi 06.
  3. Tambahkan Number Block untuk "Total Hari Cuti".Hubungkan ke Rollup SUM dari Sesi 06/07.
  4. Tambahkan Chart Block "Cuti per Jenis".Sumber data dari View yang sudah di-Group by Jenis Cuti (Sesi 08).
  5. Tambahkan Record List Block untuk pesanan/cuti berstatus "Diproses".Gunakan View yang sudah difilter dari Sesi 08.
  6. Beri judul dan rapikan tata letak Interface.Kelompokkan elemen ringkasan di atas, detail di bawah.
  7. Uji dengan menambah satu data cuti baru lewat Form.Perhatikan seluruh elemen Dashboard ikut ter-update otomatis.
Kaitan dengan proyek restoran: pola yang sama persis dipakai untuk Dashboard Pemilik pada proyek akhir — Number Block untuk Total Penjualan Hari Ini, Chart Block untuk Menu Terlaris (Group by Menu di Detail Pesanan), dan Record List untuk pesanan yang masih "Diproses".
