</dependency>

Unit Testing — Service layer dengan Mockito
1

Buat file test di src/test/java/…/service/

Satu file test per service class. Contoh: ProductServiceTest.java

2

Gunakan anotasi @ExtendWith(MockitoExtension.class)

Mock repository dengan @Mock, inject ke service dengan @InjectMocks. Jangan load Spring context — unit test harus cepat.

@ExtendWith(MockitoExtension.class)
class ProductServiceTest {
  @Mock ProductRepository repo;
  @InjectMocks ProductService service;

  @Test
  void findById_returnsProduct() {
    when(repo.findById(1L)).thenReturn(Optional.of(new Product()));
    assertNotNull(service.findById(1L));
  }
}
3

Pastikan semua service class punya test

Requirement: all service classes. Cek semua file di package service/, buat test untuk setiap method publik.

Integration Testing — Controller endpoints
1

Gunakan @SpringBootTest + MockMvc

Load full Spring context. Gunakan MockMvc untuk kirim request HTTP simulasi ke setiap endpoint controller.

@SpringBootTest
@AutoConfigureMockMvc
class ProductControllerTest {
  @Autowired MockMvc mockMvc;

  @Test
  void getProducts_returns200() throws Exception {
    mockMvc.perform(get("/api/products"))
      .andExpect(status().isOk());
  }
}
2

Gunakan Testcontainers untuk DB nyata

Tambahkan @Testcontainers dan deklarasikan container PostgreSQL agar test menggunakan DB asli, bukan H2 in-memory.

@Testcontainers
@SpringBootTest
class ProductIntegrationTest {
  @Container
  static PostgreSQLContainer<?> db =
    new PostgreSQLContainer<>("postgres:15");

  @DynamicPropertySource
  static void props(DynamicPropertyRegistry r) {
    r.add("spring.datasource.url", db::getJdbcUrl);
    r.add("spring.datasource.username", db::getUsername);
    r.add("spring.datasource.password", db::getPassword);
  }
}
3

Test semua endpoint di setiap controller

Cek: GET (list & by ID), POST (create), PUT/PATCH (update), DELETE. Verifikasi status code dan response body.

Code Coverage ≥ 70% dengan JaCoCo
1

Konfigurasi JaCoCo di pom.xml

<plugin>
  <groupId>org.jacoco</groupId>
  <artifactId>jacoco-maven-plugin</artifactId>
  <executions>
    <execution><id>prepare-agent</id><goals><goal>prepare-agent</goal></goals></execution>
    <execution>
      <id>report</id><phase>verify</phase>
      <goals><goal>report</goal></goals>
    </execution>
  </executions>
</plugin>
2

Jalankan dan lihat laporan

mvn clean verify

Buka target/site/jacoco/index.html di browser. Fokus pada kolom Missed Instructions di package service dan controller.

3

Target cakupan per package

service package
≥ 70%
controller package
≥ 70%

Jika masih di bawah 70%, tambah test case untuk branch/condition yang belum ter-cover (lihat highlight merah di laporan HTML JaCoCo).

Screenshot laporan JaCoCo (HTML) sebagai bukti submission. Pastikan nama package dan persentase coverage terlihat jelas.
Load Benchmarking — k6 (rekomendasi) atau JMeter
1

Install k6

winget install k6 # Windows
brew install k6 # macOS
2

Buat script load-test.js — simulasi user journey

import http from 'k6/http';
import { sleep } from 'k6';

export const options = {
  vus: 50, // 50 concurrent users
  duration: '5m', // 5 menit
};

export default function () {
  // 1. Login
  const loginRes = http.post('/api/auth/login', JSON.stringify({...}));
  const token = loginRes.json('token');

  // 2. Fetch list
  http.get('/api/products', { headers: { Authorization: `Bearer ${token}` } });

  // 3. Create resource
  http.post('/api/products', JSON.stringify({...}));

  // 4. Logout
  http.post('/api/auth/logout');
  sleep(1);
}
3

Jalankan dan simpan hasil

k6 run --out json=result.json load-test.js

Atau gunakan JMeter dan export .jtl report.

Pass threshold yang harus terpenuhi
API throughput≥ 100 req/s @ 50 VU
p95 latency< 500ms
Error rate< 1%
DB querytidak ada query > 200ms
JVM memorytidak heap exhaustion selama 5 menit
Untuk DB query performance, aktifkan slow query log di PostgreSQL atau gunakan Spring Actuator endpoint /actuator/metrics sambil load test berjalan.
Checklist bukti yang perlu dikumpulkan
  • Screenshot laporan JaCoCo (HTML) — pastikan ≥ 70% line coverage di package service dan controller terlihat
  • Output mvn test atau mvn verify di terminal — semua test hijau (BUILD SUCCESS)
  • File result.json dari k6 atau file .jtl dari JMeter
  • Screenshot summary k6 di terminal (atau JMeter Summary Report) yang menampilkan throughput, p95 latency, dan error rate
  • Kode test (file *Test.java) tersedia di repository GitHub
  • Script load test (load-test.js atau JMeter .jmx) juga tersedia di repo
