Bab 1
Pengenalan SQL & Database

Sebelum menulis kode SQL, kita perlu memahami konsep dasarnya terlebih dahulu.

Apa itu Database?

Database adalah tempat menyimpan data secara terstruktur. Bayangkan database seperti sebuah lemari arsip: di dalam lemari ada banyak laci (tabel), di dalam laci ada banyak dokumen (baris/row), dan setiap dokumen memiliki kolom informasi (kolom/column).

Anatomi Tabel SQL

Sebuah tabel terdiri dari kolom (mendefinisikan jenis data) dan baris (data aktual). Contoh tabel mahasiswa:

Tabel: mahasiswa
idnimnamaprodiangkatanipk
122001Budi SantosoInformatika20223.75
222002Sari DewiSistem Informasi20223.82
322003Ahmad FauziInformatika20233.60
422004Rina KusumaSistem Informasi20233.91
522005Doni PrasetyoTeknik Elektro20223.45

Membuat Database & Tabel

Perintah CREATE DATABASE membuat database baru, sedangkan CREATE TABLE membuat tabel di dalamnya.

SQL
-- Membuat database baru
CREATE DATABASE kampus;

-- Gunakan database tersebut
USE kampus; -- MySQL/MariaDB
-- atau di PostgreSQL: \c kampus

-- Membuat tabel mahasiswa
CREATE TABLE mahasiswa (
  id        SERIAL PRIMARY KEY,
  nim       VARCHAR(20)  NOT NULL UNIQUE,
  nama      VARCHAR(100) NOT NULL,
  prodi     VARCHAR(50),
  angkatan  INTEGER,
  ipk       DECIMAL(3,2)
);
ℹ️

Tipe data umum: INTEGER untuk bilangan bulat, VARCHAR(n) untuk teks (maks n karakter), DECIMAL(p,s) untuk desimal, DATE untuk tanggal, BOOLEAN untuk benar/salah.

Perintah SQL: DDL vs DML

DDL — Definisi Struktur
CREATE buat tabel/database
ALTER ubah struktur tabel
DROP hapus tabel/database
DML — Manipulasi Data
SELECT ambil data
INSERT tambah data
UPDATE ubah data
DELETE hapus data
🧠 Latihan Soal
Perintah SQL manakah yang digunakan untuk membuat tabel baru?
✅ Benar! CREATE TABLE adalah perintah DDL untuk membuat tabel baru. Diikuti nama tabel dan definisi kolom dalam kurung.
❌ Belum tepat. Perintah yang benar adalah CREATE TABLE nama_tabel. Kata kunci CREATE digunakan untuk membuat objek baru di database.

SELECT — Mengambil Data

Perintah SELECT adalah yang paling sering digunakan dalam SQL. Fungsinya untuk mengambil (membaca) data dari tabel.

SELECT Semua Kolom

Tanda bintang (*) berarti "ambil semua kolom".

SQL
SELECT * FROM mahasiswa;

Alias Kolom dengan AS

Gunakan AS untuk memberi nama tampilan berbeda pada kolom tanpa mengubah nama aslinya di database.

SQL

  nama            AS "Nama Mahasiswa",
  prodi           AS "Program Studi",
  ipk             AS "Indeks Prestasi"

DISTINCT — Nilai Unik

Gunakan DISTINCT untuk menghilangkan duplikat dalam hasil query.

SQL
-- Tampilkan daftar prodi yang ada (tanpa duplikat)
SELECT DISTINCT prodi
FROM mahasiswa;

LIMIT — Batasi Jumlah Baris

SQL
-- Tampilkan hanya 3 baris pertama
SELECT * FROM mahasiswa
LIMIT 3;
💡

Penulisan SQL tidak case-sensitive. SELECT sama dengan select. Namun, kebiasaan baik adalah menulis kata kunci SQL dengan HURUF KAPITAL agar mudah dibedakan dari nama tabel/kolom.

🧠 Latihan Soal
Apa fungsi kata kunci DISTINCT dalam perintah SELECT?
✅ Tepat sekali! DISTINCT menghilangkan baris duplikat sehingga setiap nilai hanya muncul satu kali dalam hasil query.
❌ Belum tepat. DISTINCT digunakan untuk menghilangkan nilai duplikat. Misalnya SELECT DISTINCT prodi hanya menampilkan satu baris untuk setiap program studi yang unik.

Bab 3
Filter dengan WHERE & Pengurutan ORDER BY

Jarang kita ingin melihat semua data sekaligus. Dengan WHERE kita bisa menyaring data, dan ORDER BY untuk mengurutkannya.

WHERE — Menyaring Data

Tuliskan kondisi setelah WHERE. Hanya baris yang memenuhi kondisi yang akan ditampilkan.

SQL
-- Tampilkan mahasiswa dari prodi Informatika
SELECT nama, prodi, ipk
FROM   mahasiswa
WHERE  prodi = 'Informatika';
Hasil
namaprodiipk
Budi SantosoInformatika3.75
Ahmad FauziInformatika3.60

Operator Perbandingan

=
Sama dengan
WHERE ipk = 3.75
<> atau !=
Tidak sama dengan
WHERE prodi <> 'Informatika'
> dan <
Lebih besar / lebih kecil
WHERE ipk > 3.70
>= dan <=
Lebih besar/kecil atau sama
WHERE angkatan >= 2022

AND, OR, NOT

Gabungkan beberapa kondisi dengan operator logika.

SQL
-- AND: semua kondisi harus terpenuhi
SELECT nama, prodi, ipk
FROM   mahasiswa
WHERE  prodi = 'Informatika'
  AND  ipk > 3.60;

-- OR: salah satu kondisi terpenuhi
SELECT nama, prodi
FROM   mahasiswa
WHERE  prodi = 'Informatika'
  OR   prodi = 'Teknik Elektro';

IN — Cocokkan dari Daftar

SQL
-- IN lebih ringkas daripada banyak OR
SELECT nama, prodi
FROM   mahasiswa
WHERE  prodi IN ('Informatika', 'Teknik Elektro');

BETWEEN — Rentang Nilai

SQL
-- IPK antara 3.60 dan 3.80 (inklusif)
SELECT nama, ipk
FROM   mahasiswa
WHERE  ipk BETWEEN 3.60 AND 3.80;

LIKE — Pencarian Pola Teks

% menggantikan nol atau lebih karakter. _ menggantikan tepat satu karakter.

SQL
-- Nama yang diawali huruf 'B'
SELECT nama FROM mahasiswa
WHERE  nama LIKE 'B%';

-- Nama yang mengandung kata 'Dewi'
SELECT nama FROM mahasiswa
WHERE  nama LIKE '%Dewi%';

ORDER BY — Mengurutkan Hasil

ASC (ascending = kecil ke besar, default) dan DESC (descending = besar ke kecil).

SQL
-- Urutkan berdasarkan IPK tertinggi dulu
SELECT nama, prodi, ipk
FROM   mahasiswa
ORDER BY ipk DESC;

-- Urutkan berdasarkan prodi (A-Z), lalu IPK tertinggi
SELECT nama, prodi, ipk
FROM   mahasiswa
ORDER BY prodi ASC, ipk DESC;
Hasil — Urut IPK DESC
namaprodiipk
Rina KusumaSistem Informasi3.91
Sari DewiSistem Informasi3.82
Budi SantosoInformatika3.75
Ahmad FauziInformatika3.60
Doni PrasetyoTeknik Elektro3.45
🧠 Latihan Soal
Query manakah yang menampilkan mahasiswa dengan IPK di atas 3.70 dari prodi Informatika?
✅ Benar! Gunakan AND ketika kedua kondisi harus terpenuhi secara bersamaan. OR dipakai jika salah satu kondisi sudah cukup.
❌ Kurang tepat. Karena kedua kondisi harus terpenuhi sekaligus (IPK > 3.70 dan prodi Informatika), gunakan operator AND. OR akan menampilkan mahasiswa yang memenuhi salah satu kondisi saja.

Bab 4
Manipulasi Data: INSERT, UPDATE, DELETE

Tiga perintah inti untuk mengubah isi data di dalam tabel.

INSERT INTO — Menambah Data

SQL
-- Menyebutkan kolom (disarankan)
INSERT INTO mahasiswa (nim, nama, prodi, angkatan, ipk)
VALUES ('22006', 'Fitri Amalia', 'Informatika', 2022, 3.88);

-- Insert banyak baris sekaligus
INSERT INTO mahasiswa (nim, nama, prodi, angkatan, ipk)
VALUES
  ('22007', 'Hendra Wijaya', 'Teknik Elektro', 2023, 3.55),
  ('22008', 'Maya Putri',    'Informatika',    2023, 3.72);
💡

Selalu sebutkan nama kolom saat INSERT. Jika urutan kolom di tabel berubah di kemudian hari, query INSERT yang menyebutkan kolom tetap bekerja dengan benar.

UPDATE — Mengubah Data

Selalu sertakan WHERE saat UPDATE, agar tidak mengubah semua baris secara tidak sengaja!

SQL
-- Update IPK mahasiswa dengan NIM 22001
UPDATE mahasiswa
SET    ipk = 3.80
WHERE  nim = '22001';

-- Update beberapa kolom sekaligus
UPDATE mahasiswa
SET    prodi = 'Informatika',
       angkatan = 2023
WHERE  id = 5;
⛔

Berbahaya! Jika kamu menulis UPDATE mahasiswa SET ipk = 0 tanpa WHERE, semua baris di tabel mahasiswa akan ter-update! Selalu cek kondisi WHERE sebelum menjalankan UPDATE.

DELETE — Menghapus Data

SQL
-- Hapus satu baris berdasarkan id
DELETE FROM mahasiswa
WHERE  id = 5;

-- Hapus semua mahasiswa angkatan 2020
DELETE FROM mahasiswa
WHERE  angkatan = 2020;
⚠️

Tips aman: Sebelum menjalankan DELETE atau UPDATE, ubah dulu jadi SELECT dengan kondisi WHERE yang sama untuk memastikan baris yang akan terpengaruh sudah benar. Setelah yakin, baru ganti SELECT ke DELETE/UPDATE.

ALTER TABLE — Mengubah Struktur Tabel

SQL
-- Menambah kolom baru
ALTER TABLE mahasiswa
ADD COLUMN email VARCHAR(100);

-- Mengubah tipe data kolom
ALTER TABLE mahasiswa
ALTER COLUMN angkatan TYPE SMALLINT; -- PostgreSQL

-- Menghapus kolom
ALTER TABLE mahasiswa
DROP COLUMN email;
🧠 Latihan Soal
Perintah SQL manakah yang aman untuk mengubah IPK menjadi 3.95 hanya untuk mahasiswa bernama Sari Dewi?
✅ Benar! UPDATE ... SET ... WHERE adalah pola yang tepat. Klausa WHERE memastikan hanya baris Sari Dewi yang berubah, bukan seluruh tabel.
❌ Belum tepat. Jawaban A berbahaya karena mengubah semua baris. Pola yang benar adalah UPDATE tabel SET kolom = nilai WHERE kondisi. Tidak ada perintah CHANGE di SQL standar.

Bab 5
Fungsi Agregasi & GROUP BY

Fungsi agregasi melakukan perhitungan pada sekumpulan baris dan mengembalikan satu nilai. Sering dipakai bersama GROUP BY.

Fungsi Agregasi Dasar

SQL
-- COUNT: hitung jumlah baris
SELECT COUNT(*) AS total_mahasiswa
FROM   mahasiswa;

-- SUM: jumlah total (biasanya untuk angka)
SELECT SUM(ipk) AS jumlah_ipk
FROM   mahasiswa;

-- AVG: rata-rata
SELECT ROUND(AVG(ipk), 2) AS rata_ipk
FROM   mahasiswa;

-- MIN dan MAX
SELECT
  MIN(ipk) AS ipk_terendah,
  MAX(ipk) AS ipk_tertinggi
FROM mahasiswa;
Hasil COUNT, AVG, MIN, MAX
total_mahasiswarata_ipkipk_terendahipk_tertinggi
53.713.453.91

GROUP BY — Kelompokkan Data

Gunakan GROUP BY untuk mengelompokkan baris berdasarkan nilai kolom tertentu, lalu jalankan fungsi agregasi per kelompok.

SQL
-- Hitung jumlah mahasiswa dan rata-rata IPK per prodi
SELECT
  prodi,
  COUNT(*) AS jumlah,
  ROUND(AVG(ipk), 2) AS rata_ipk
FROM   mahasiswa
GROUP BY prodi
ORDER BY jumlah DESC;
Hasil
prodijumlahrata_ipk
Informatika23.68
Sistem Informasi23.87
Teknik Elektro13.45

HAVING — Filter Setelah GROUP BY

WHERE tidak bisa dipakai untuk memfilter hasil agregasi. Gunakan HAVING sebagai gantinya.

SQL
-- Tampilkan prodi yang rata-rata IPK-nya > 3.70
SELECT
  prodi,
  ROUND(AVG(ipk), 2) AS rata_ipk
FROM   mahasiswa
GROUP BY prodi
HAVING   AVG(ipk) > 3.70
ORDER BY rata_ipk DESC;
Hasil
prodirata_ipk
Sistem Informasi3.87
Informatika3.68
ℹ️

Urutan eksekusi SQL: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT. Ini menjelaskan mengapa HAVING (setelah GROUP BY) bisa memfilter hasil agregasi, sedangkan WHERE tidak bisa.

🧠 Latihan Soal
Kamu ingin menampilkan prodi yang memiliki lebih dari 1 mahasiswa. Klausa mana yang tepat digunakan?
✅ Benar! HAVING COUNT(*) > 1 memfilter hasil GROUP BY. WHERE tidak bisa digunakan untuk fungsi agregasi seperti COUNT.
❌ Belum tepat. WHERE tidak bisa digunakan untuk memfilter hasil fungsi agregasi. Gunakan HAVING setelah GROUP BY untuk memfilter berdasarkan nilai COUNT, AVG, SUM, dll.

JOIN — Menggabungkan Tabel

Data nyata disimpan di beberapa tabel yang saling terhubung. JOIN menggabungkan baris dari dua tabel atau lebih berdasarkan kolom yang berelasi.

Contoh: Dua Tabel Berelasi

Misalnya kita memiliki tabel nilai yang menyimpan nilai mata kuliah mahasiswa:

Tabel: nilai

32Pemrograman Web3A
43Basis Data3B+
54Jaringan Komputer2A

INNER JOIN — Baris yang Cocok di Kedua Tabel

Menampilkan hanya baris yang memiliki pasangan di kedua tabel.

SQL
SELECT
  m.nama,
  m.prodi,
  n.matkul,
  n.nilai
FROM       mahasiswa m
INNER JOIN nilai     n ON m.id = n.mahasiswa_id
ORDER BY   m.nama;

LEFT JOIN — Semua dari Tabel Kiri

Menampilkan semua baris dari tabel kiri, meski tidak ada pasangan di tabel kanan. Nilai kolom kanan akan NULL.

SQL
-- Tampilkan SEMUA mahasiswa, termasuk yang belum ada nilai
SELECT
  m.nama,
  n.matkul,
  n.nilai
FROM      mahasiswa m
LEFT JOIN nilai     n ON m.id = n.mahasiswa_id;

ℹ️

Doni Prasetyo muncul dengan nilai NULL karena ia belum memiliki data di tabel nilai. INNER JOIN tidak akan menampilkan Doni sama sekali. Gunakan LEFT JOIN jika kamu butuh "semua data dari tabel utama, meski tidak ada relasi".

JOIN dengan WHERE dan GROUP BY

SQL
-- Hitung jumlah mata kuliah per mahasiswa
SELECT
  m.nama,
  m.prodi,
  COUNT(n.id) AS jumlah_matkul
FROM       mahasiswa m
LEFT JOIN  nilai     n ON m.id = n.mahasiswa_id
GROUP BY   m.id, m.nama, m.prodi
ORDER BY   jumlah_matkul DESC;
🧠 Latihan Soal
Kamu ingin menampilkan semua mahasiswa termasuk yang belum punya nilai. JOIN mana yang tepat?
✅ Tepat! LEFT JOIN mempertahankan semua baris dari tabel kiri (mahasiswa), dan mengisi kolom dari tabel kanan (nilai) dengan NULL jika tidak ada kecocokan.
❌ Belum tepat. INNER JOIN hanya menampilkan baris yang ada di kedua tabel, sehingga mahasiswa tanpa nilai tidak akan muncul. Gunakan LEFT JOIN untuk mempertahankan semua baris dari tabel kiri.

SQL Cheat Sheet

Ringkasan semua perintah SQL yang sudah dipelajari. Simpan halaman ini untuk referensi cepat.

Struktur Query SELECT Lengkap

Urutan Penulisan
SELECT   kolom1, COUNT(*), AVG(kolom2)   -- 1. Pilih kolom / fungsi
FROM     nama_tabel                         -- 2. Dari tabel mana
JOIN     tabel_lain ON kondisi_join        -- 3. Gabungkan tabel (opsional)
WHERE    kondisi_baris                      -- 4. Filter sebelum group
GROUP BY kolom1                            -- 5. Kelompokkan (opsional)
HAVING   kondisi_agregat                   -- 6. Filter setelah group
ORDER BY kolom1 ASC, kolom2 DESC          -- 7. Urutan hasil
LIMIT    10;                               -- 8. Batas jumlah baris
