1

Registrasi & Membuat Project

1
Buka halaman Neon.tech

Kunjungi neon.tech lalu klik tombol Sign Up. Kamu bisa mendaftar menggunakan akun Google atau GitHub — tidak perlu kartu kredit untuk plan gratis (Free Tier).

2
Buat project baru

Setelah login, klik New Project. Isi nama project dan pilih region terdekat. Untuk pengguna Indonesia, pilih Asia Pacific (Singapore) agar latensi lebih rendah. Klik Create Project.

ℹ️

Free Tier Neon menyediakan 0.5 GB storage, 1 compute unit, database branching, dan akses penuh ke fitur PostgreSQL. Cocok untuk development, tugas kuliah, dan prototyping.

3
Simpan Connection String

Setelah project dibuat, Neon langsung menampilkan connection string. Salin dan simpan di tempat yang aman — kamu akan membutuhkannya untuk koneksi ke DBeaver maupun aplikasi.

Contoh format connection string
postgresql://username:password@ep-xxxxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require

Membuat Database dan Tabel

1
Buka SQL Editor di dashboard

Di sidebar kiri Neon Dashboard, klik SQL Editor. Ini adalah query editor berbasis web yang langsung terhubung ke database kamu — tanpa perlu install software apapun.

2
Buat tabel baru

Ketik perintah SQL berikut lalu klik tombol Run atau tekan Ctrl+Enter:

SQL — Membuat tabel mahasiswa
CREATE TABLE mahasiswa ( id SERIAL PRIMARY KEY, nim VARCHAR(20) NOT NULL UNIQUE, nama VARCHAR(100) NOT NULL, prodi VARCHAR(50), angkatan INTEGER, ipk DECIMAL(3,2) );
3
Verifikasi tabel berhasil dibuat

Jalankan query berikut untuk memastikan tabel sudah terdaftar dalam schema public:

SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
💡

Neon mendukung semua fitur PostgreSQL standar: views, functions, triggers, foreign keys, dan extensions seperti pgvector untuk kebutuhan AI/ML, postgis untuk geospasial, dll.

Upload Data ke Neon

Ada tiga cara untuk mengimpor data ke Neon. Pilih metode yang paling sesuai dengan kebutuhanmu:

1
Insert data via SQL Editor

Cara paling sederhana — cocok untuk jumlah data kecil. Buka SQL Editor di dashboard Neon dan jalankan:

SQL — INSERT data
INSERT INTO mahasiswa (nim, nama, prodi, angkatan, ipk) VALUES ('22001', 'Budi Santoso', 'Informatika', 2022, 3.75), ('22002', 'Sari Dewi', 'Sistem Informasi', 2022, 3.82), ('22003', 'Ahmad Fauzi', 'Informatika', 2022, 3.60), ('22004', 'Rina Kusuma', 'Sistem Informasi', 2022, 3.91);
2
Verifikasi data masuk

Setelah INSERT berhasil, jalankan query SELECT untuk mengecek data:

SELECT * FROM mahasiswa ORDER BY nim;
1
Siapkan file CSV

Buat file mahasiswa.csv dengan format berikut. Baris pertama adalah header yang harus sama persis dengan nama kolom tabel:

mahasiswa.csv
nim,nama,prodi,angkatan,ipk 22001,Budi Santoso,Informatika,2022,3.75 22002,Sari Dewi,Sistem Informasi,2022,3.82 22003,Ahmad Fauzi,Informatika,2022,3.60 22004,Rina Kusuma,Sistem Informasi,2022,3.91
2
Install psql di komputer lokal

Pastikan PostgreSQL client sudah terinstall. Di Windows: download installer dari postgresql.org. Di Ubuntu/WSL:

sudo apt install postgresql-client
3
Jalankan perintah \COPY via psql

Ganti connection string dengan milikmu, lalu jalankan di terminal/command prompt:

psql "postgresql://user:password@host/neondb?sslmode=require" \ -c "\COPY mahasiswa(nim,nama,prodi,angkatan,ipk) \ FROM 'mahasiswa.csv' CSV HEADER DELIMITER ',';"
⚠️

Gunakan \COPY (dengan backslash) — bukan COPY. Perintah \COPY membaca file dari komputer lokal (sisi client), sedangkan COPY akan mencari file di server Neon yang tidak kita miliki aksesnya.

💡

Alternatif via DBeaver: Klik kanan tabel → Import Data → pilih file CSV. DBeaver akan memetakan kolom secara otomatis — cara termudah tanpa perlu command line.

1
Ekspor database lama dengan pg_dump

Jika ingin migrasi dari PostgreSQL lain (lokal, Supabase, Railway, dsb.), buat dump file terlebih dahulu:

pg_dump "postgresql://user:pass@old-host/dbname" \ --no-acl --no-owner -Fc -f backup.dump
2
Restore ke Neon menggunakan pg_restore

Gunakan connection string Neon sebagai target restore:

pg_restore \ -d "postgresql://user:pass@neon-host/neondb?sslmode=require" \ --no-acl --no-owner -Fc backup.dump
ℹ️

Metode ini ideal untuk migrasi database berukuran besar. Neon mendukung pg_restore versi 14 ke atas. Neon saat ini menjalankan PostgreSQL 16 — pastikan versi pg_dump tidak lebih baru dari versi Neon.

3
Verifikasi hasil restore

Setelah restore selesai, cek semua tabel sudah ada dengan perintah:

psql "postgresql://user:pass@neon-host/neondb?sslmode=require" -c "\dt"
4

Koneksi ke DBeaver

1
Install DBeaver Community

Download DBeaver Community Edition (gratis) dari dbeaver.io. Tersedia untuk Windows, macOS, dan Linux. Versi Community sudah cukup untuk semua kebutuhan PostgreSQL.

2
Buat koneksi baru

Buka DBeaver → klik ikon plug (New Database Connection) di toolbar, atau gunakan menu Database → New Database Connection, atau tekan Ctrl+Shift+N. Pilih PostgreSQL dari daftar, lalu klik Next.

3
Isi parameter koneksi

Salin detail berikut dari halaman Connection Details di Neon Dashboard project kamu:

🖥 Host
ep-xxxxx.ap-southeast-1.aws.neon.tech
# Port
5432
🗄 Database
neondb
👤 Username
[nama_user_neon]
🔑 Password
••••••••••••••
4
Aktifkan SSL — LANGKAH WAJIB

Di jendela koneksi DBeaver, klik tab SSL (di sebelah tab Main). Centang opsi Use SSL, lalu ubah SSL Mode menjadi require. Tanpa langkah ini koneksi pasti gagal.

⚠️

SSL wajib diaktifkan. Jika kamu melewati langkah ini, DBeaver akan menampilkan error "SSL connection is required" atau "connection refused". Pastikan SSL Mode sudah diset ke require sebelum klik Test Connection.

5
Test Connection & Finish

Klik tombol Test Connection... di bagian bawah. Jika muncul pop-up hijau dengan tanda centang dan tulisan "Connected", berarti berhasil. Klik Finish untuk menyimpan konfigurasi koneksi.

6
Download driver PostgreSQL (jika diminta)

Jika ini pertama kali menggunakan PostgreSQL di DBeaver, akan muncul dialog untuk men-download driver JDBC. Klik Download dan tunggu selesai — ini otomatis dan hanya perlu dilakukan sekali.

7
Jelajahi database

Di panel Database Navigator (kiri), expand koneksi kamu → Databases → neondb → Schemas → public → Tables. Klik dua kali pada tabel untuk melihat data, atau klik kanan untuk opsi edit, export, import, dan lain-lain.

💡

Import CSV via DBeaver: Klik kanan tabel → Import Data → pilih format CSV → pilih file → petakan kolom → klik Start. Ini cara termudah untuk upload data tanpa perlu command line sama sekali.

5

Tips & Troubleshooting

⏸ Compute auto-suspend (cold start)

Neon Free Tier mematikan compute secara otomatis setelah 5 menit tidak ada aktivitas. Koneksi pertama setelah idle akan terasa lambat (~1–3 detik) karena compute perlu "bangun". Ini normal dan disebut cold start — tidak ada yang rusak.

🔌 Error: "endpoint is disabled"

Buka Neon Dashboard → Project → Settings → Compute → pastikan compute tidak di-disable secara manual. Jika melampaui kuota Free Tier, compute bisa terkunci hingga bulan berikutnya.

🔀 Gunakan Branching untuk dev vs prod

Neon mendukung database branching seperti Git. Buat branch dev dari branch main untuk eksperimen tanpa risiko merusak data produksi. Branch dibuat secara instan (copy-on-write) tanpa menyalin data secara fisik.

🔗 Connection Pooling untuk aplikasi web

Neon menyediakan URL khusus dengan PgBouncer untuk connection pooling. Di dashboard pilih Pooled connection — lebih efisien untuk aplikasi Next.js/Node.js yang membuka banyak koneksi paralel. Untuk DBeaver, selalu gunakan URL direct (non-pooled).

⏰ Timeout koneksi di DBeaver

Jika DBeaver sering disconnect setelah idle, buka Edit Connection → Connection settings → Connection → Keep-Alive dan centang Keep-Alive. Set interval ke 60 detik untuk menjaga koneksi tetap aktif.

Perintah psql berguna untuk Neon
# Simpan connection string sebagai variabel (Linux/Mac) export NEON="postgresql://user:pass@host/neondb?sslmode=require" # Cek semua tabel dalam schema public psql "$NEON" -c "\dt" # Cek ukuran database psql "$NEON" -c "SELECT pg_size_pretty(pg_database_size(current_database()));" # Cek semua ekstensi yang terpasang psql "$NEON" -c "\dx" # Export tabel ke file CSV psql "$NEON" -c "\COPY mahasiswa TO 'output.csv' CSV HEADER;" # Import CSV ke tabel yang sudah ada psql "$NEON" -c "\COPY mahasiswa FROM 'input.csv' CSV HEADER;"
