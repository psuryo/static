Kunci Jawaban — 10 Soal SQL
Jawaban referensi; variasi query yang menghasilkan output sama juga diterima
💡

Query di bawah adalah jawaban referensi. Peserta boleh menggunakan pendekatan lain (alias berbeda, urutan JOIN berbeda, dsb.) selama hasil yang dihasilkan benar.

1
SELECT · WHERE · ORDER BY
● Mudah
Customer dari kota Yogyakarta, diurutkan alfabetis.
✅ Hasil: 2 baris (Andi Setiawan, Eko Nugroho)
SQL — Jawaban No. 1
SELECT *
FROM   customer
WHERE  kota = 'Yogyakarta'
ORDER BY nama ASC;
Penjelasan: WHERE kota = 'Yogyakarta' memfilter baris yang sesuai. ORDER BY nama ASC mengurutkan hasil dari A–Z (ASC bisa dihilangkan karena sudah default).
2
SELECT · WHERE · BETWEEN · ORDER BY
● Mudah
Barang dengan harga Rp100.000–Rp500.000, termurah lebih dulu.
✅ Hasil: 4 baris (Mouse Wireless, Buku SQL Lanjutan, Buku Algoritma, Tas Ransel)
SQL — Jawaban No. 2
SELECT nama_barang, harga
FROM   barang
WHERE  harga BETWEEN 100000 AND 500000
ORDER BY harga ASC;
Penjelasan: BETWEEN a AND b setara dengan harga >= 100000 AND harga <= 500000 (inklusif di kedua ujung). Alternatif menggunakan WHERE dengan dua kondisi juga benar.
3
UPDATE · SET · WHERE
● Mudah
Kurangi stok Flash Disk 64GB (id=6) sebanyak 10 buah (dari 100 menjadi 90).
✅ Hasil: stok id=6 menjadi 90
SQL — Jawaban No. 3
UPDATE barang
SET    stok = stok - 10
WHERE  id = 6;

-- Verifikasi hasil:
SELECT nama_barang, stok FROM barang WHERE id = 6;
Penjelasan: Menggunakan stok = stok - 10 lebih aman daripada menulis nilai tetap (stok = 90) karena tidak tergantung nilai awal yang dihapal. WHERE wajib ada — tanpa WHERE, seluruh baris akan diperbarui!
4
SELECT · WHERE · LIKE
● Mudah
Barang yang namanya mengandung kata "Buku".
✅ Hasil: 2 baris (Buku Algoritma, Buku SQL Lanjutan)
SQL — Jawaban No. 4
SELECT nama_barang, kategori, harga
FROM   barang
WHERE  nama_barang LIKE '%Buku%';
Penjelasan: % adalah wildcard yang cocok dengan nol atau lebih karakter. Pola '%Buku%' menemukan "Buku" di posisi mana pun dalam string. Di MySQL, LIKE tidak membedakan huruf besar/kecil secara default.
5
INNER JOIN ×2 · ORDER BY
● Menengah
Riwayat transaksi lengkap: tanggal, nama customer, nama barang, jumlah — terbaru lebih dulu.
✅ Hasil: 10 baris
SQL — Jawaban No. 5
SELECT
  t.tgl_transaksi,
  c.nama        AS nama_customer,
  b.nama_barang,
  t.jumlah
FROM   transaksi t
INNER JOIN customer c ON t.customer_id = c.id
INNER JOIN barang   b ON t.barang_id   = b.id
ORDER BY t.tgl_transaksi DESC;
Penjelasan: Dua INNER JOIN menghubungkan tiga tabel sekaligus melalui foreign key. Alias tabel (t, c, b) membuat query lebih ringkas dan menghindari ambiguitas nama kolom. ORDER BY DESC menampilkan transaksi terbaru di atas.
6
JOIN · Kalkulasi Kolom
● Menengah
Total nilai pembelian per transaksi (harga × jumlah).
✅ Hasil: 10 baris dengan kolom total_harga
SQL — Jawaban No. 6
SELECT
  c.nama              AS nama_customer,
  b.nama_barang,
  t.jumlah,
  b.harga             AS harga_satuan,
  b.harga * t.jumlah  AS total_harga
FROM   transaksi t
INNER JOIN customer c ON t.customer_id = c.id
INNER JOIN barang   b ON t.barang_id   = b.id
ORDER BY t.id;
Penjelasan: Ekspresi b.harga * t.jumlah membuat kolom kalkulasi yang dihitung langsung saat query dijalankan — kolom ini tidak tersimpan di tabel. Alias total_harga memberikan nama yang bermakna pada hasil kalkulasi.
7
JOIN · GROUP BY · SUM · ORDER BY
● Menengah
Total belanja per customer, terbesar lebih dulu.
✅ Hasil: 5 baris (hanya customer yang pernah transaksi)
SQL — Jawaban No. 7
SELECT
  c.nama                           AS nama_customer,
  SUM(b.harga * t.jumlah)           AS total_belanja
FROM   transaksi t
INNER JOIN customer c ON t.customer_id = c.id
INNER JOIN barang   b ON t.barang_id   = b.id
GROUP BY c.id, c.nama
ORDER BY total_belanja DESC;
Penjelasan: GROUP BY c.id, c.nama mengelompokkan semua transaksi milik customer yang sama. SUM() menjumlahkan nilai setiap transaksi dalam kelompok tersebut. Menyertakan c.id di GROUP BY mencegah masalah jika ada dua customer dengan nama sama.
8
LEFT JOIN · IS NULL
● Menengah
Customer yang belum pernah bertransaksi sama sekali.
✅ Hasil: 1 baris (Fitri Handayani — id=6)
SQL — Jawaban No. 8
SELECT
  c.nama,
  c.email
FROM      customer  c
LEFT JOIN transaksi t ON c.id = t.customer_id
WHERE t.id IS NULL;
Penjelasan: LEFT JOIN mempertahankan semua baris dari tabel kiri (customer), bahkan yang tidak punya pasangan di transaksi. Baris tanpa pasangan akan memiliki nilai NULL di kolom-kolom transaksi. Filter WHERE t.id IS NULL menyisakan hanya customer yang tidak pernah transaksi.
9
JOIN · GROUP BY · SUM · HAVING
● Sulit
Kategori barang dengan total satuan terjual > 5.
✅ Hasil: 2 baris (Elektronik: 10, Buku: 7)
SQL — Jawaban No. 9
SELECT
  b.kategori,
  SUM(t.jumlah) AS total_satuan_terjual
FROM   transaksi t
INNER JOIN barang b ON t.barang_id = b.id
GROUP BY b.kategori
HAVING   SUM(t.jumlah) > 5
ORDER BY total_satuan_terjual DESC;
Penjelasan: Perbedaan kunci: WHERE memfilter baris sebelum agregasi, sedangkan HAVING memfilter sesudah agregasi. Karena kondisi > 5 diterapkan pada hasil SUM(), harus menggunakan HAVING, bukan WHERE.
10
JOIN · GROUP BY · ORDER BY · LIMIT
● Sulit
Customer dengan total belanja tertinggi (1 baris saja).
✅ Hasil: 1 baris — Andi Setiawan, Yogyakarta, Rp8.800.000 (1 Laptop + 2 Mouse + 2 Buku Algoritma)
SQL — Jawaban No. 10
SELECT
  c.nama                           AS nama_customer,
  c.kota,
  SUM(b.harga * t.jumlah)           AS total_belanja
FROM   transaksi t
INNER JOIN customer c ON t.customer_id = c.id
INNER JOIN barang   b ON t.barang_id   = b.id
GROUP BY c.id, c.nama, c.kota
ORDER BY total_belanja DESC
LIMIT  1;
Penjelasan: Ini adalah pengembangan dari Soal 7 dengan dua tambahan: (1) kolom c.kota disertakan di SELECT dan GROUP BY; (2) LIMIT 1 mengambil hanya baris teratas setelah diurutkan DESC, sehingga menghasilkan customer dengan belanja terbesar.
