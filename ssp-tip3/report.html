Latihan Praktikum: Desain Report Penjualan Terintegrasi

A. Tujuan Pembelajaran

  • Mahasiswa mampu mengimplementasikan relasi antar tabel (Entity Relationship) di Microsoft Access.
  • Mahasiswa mampu membuat dan menggunakan Query untuk menggabungkan data dari berbagai tabel dan membuat Calculated Field.
  • Mahasiswa mampu mendesain Report dengan fitur Grouping dan Summary/Total.

B. Skenario & Dataset

Sebuah toko membutuhkan laporan penjualan bulanan yang dikelompokkan berdasarkan pelanggan (Customer). Anda diminta menggunakan 3 tabel berikut untuk menghasilkan satu laporan utuh.

C. Langkah Pengerjaan

Langkah 1: Relasi Tabel (Database Tools -> Relationships)

Tarik garis relasi (Enforce Referential Integrity) antara:

  • ID pada tbl_customer ke Customer_ID pada tbl_transaksi (One-to-Many).
  • ID pada tbl_barang ke Barang_ID pada tbl_transaksi (One-to-Many).

Langkah 2: Pembuatan Query (Create -> Query Design)

Buat query baru dengan nama qry_rekap_penjualan. Masukkan ketiga tabel ke dalam workspace. Tarik field berikut ke grid di bawah:

  1. Nama (dari tbl_customer)
  2. Tgl_Transaksi (dari tbl_transaksi)
  3. Nama_Barang (dari tbl_barang)
  4. Harga (dari tbl_barang)
  5. Jumlah (dari tbl_transaksi)

Buat field kalkulasi baru pada kolom kosong dengan format sintaks Access berikut:
Total_Harga: [Harga]*[Jumlah]

Langkah 3: Pembuatan Report (Create -> Report Wizard)

Gunakan qry_rekap_penjualan sebagai sumber data report Anda.

  • Fields: Pindah semua field (>>).
  • Grouping: Lakukan Grouping berdasarkan Nama (Customer).
  • Sorting: Urutkan berdasarkan Tgl_Transaksi secara Ascending.
  • Summary Options: Centang kotak Sum pada field Total_Harga untuk menampilkan total belanja per customer.
  • Layout: Pilih Stepped atau Block, orientasi Portrait atau Landscape sesuai kebutuhan ruang.

Simpan dengan nama rpt_penjualan_customer.

Langkah 4: Formatting (Design View)

Masuk ke Design View pada Report yang baru dibuat. Lakukan penyesuaian User Interface (UI):

  • Ubah format angka pada Harga dan Total_Harga menjadi Currency (Rp).
  • Rapikan label judul (misal: "Nama_Barang" menjadi "Nama Produk").
  • Beri warna background yang berbeda pada bagian Customer Name Header agar mudah dibaca.
