Tugas: Desain Relational Database

Tujuan: Menunjukkan pemahaman Anda tentang Master Table, Transaction Table, Tipe Data (Data Types), dan Integritas Referensial (Referential Integrity) menggunakan Microsoft Access.

ATURAN PENTING PENGUMPULAN: Anda WAJIB MENUTUP TOTAL aplikasi Microsoft Access sebelum menyalin atau mengunggah file. Jika Anda mengumpulkan file yang masih terkunci (.laccdb), file tersebut tidak akan bisa dibuka saat sesi review langsung di depan kelas dan nilai otomatis dianggap nol. Kumpulkan file format asli .accdb.

Skenario: "Toko Buku Kampus"

Anda telah disewa untuk membangun sistem pelacakan data pada sebuah toko buku kampus. Toko tersebut perlu mencatat data pelanggan (mahasiswa), buku yang tersedia untuk dijual, serta transaksi pembelian yang terjadi secara riil.

Instruksi Kerja

Buatlah database baru di Microsoft Access dengan nama file TokoBuku_NamaAnda.accdb dan selesaikan tugas-tugas berikut:

Tugas 1: Membuat Master Table

Buatlah dua master table (tabel induk) di bawah ini dengan struktur yang tepat:

1. Nama Tabel: tblMahasiswa

Nama Field Tipe Data (Data Type) Jenis Kunci (Key)
StudentID AutoNumber Primary Key (PK)
NamaDepan Short Text -
NamaBelakang Short Text -

2. Nama Tabel: tblBuku

Nama Field Tipe Data (Data Type) Jenis Kunci (Key)
BookID AutoNumber Primary Key (PK)
JudulBuku Short Text -
Harga Currency -

Tugas 2: Membuat Transaction Table

Buatlah tabel transaksi untuk mencatat aktivitas pembelian. Perhatikan baik-baik perubahan tipe data pada field yang menjadi Foreign Key.

3. Nama Tabel: tblPenjualan

Nama Field Tipe Data (Data Type) Jenis Kunci (Key)
SaleID AutoNumber Primary Key (PK)
StudentID Number Foreign Key (FK)
BookID Number Foreign Key (FK)
TanggalJual Date/Time -

Tugas 3: Membuat Relasi Antar Tabel

  • Buka jendela Relationships Window di Access dan masukkan ketiga tabel di atas.
  • Hubungkan field tblMahasiswa.StudentID ke field tblPenjualan.StudentID.
  • Hubungkan field tblBuku.BookID ke field tblPenjualan.BookID.
  • Sangat Penting: Centang pilihan untuk "Enforce Referential Integrity" pada kedua jalur relasi tersebut. Garis relasi Anda harus menampilkan simbol angka 1 dan simbol infinity (∞) secara jelas.

Tugas 4: Pengisian Data & Uji Coba Sistem

  • Masukkan minimal 5 data mahasiswa pada tabel tblMahasiswa.
  • Masukkan minimal 5 data buku pada tabel tblBuku.
  • Catat minimal 10 transaksi pada tabel tblPenjualan dengan menggunakan ID yang valid (yang memang ada di tabel master).
  • Uji Stres Data (Stress Test): Cobalah memasukkan data penjualan baru dengan mengisi StudentID angka 99 (sementara mahasiswa dengan ID 99 tidak ada). Pastikan Microsoft Access menolak dan memblokir input tersebut secara otomatis.

Kriteria Penilaian Secara Langsung (Live Review)

Pada pertemuan kelas berikutnya, beberapa file mahasiswa akan dipilih secara acak untuk dibuka di proyektor guna memeriksa:

  • Ketepatan pemilihan Tipe Data (khususnya Foreign Key yang wajib diset ke 'Number', bukan 'AutoNumber').
  • Aktifnya simbol Integritas Referensial (1 dan ∞) pada jendela Relationships.
  • Kerapian database dan tidak adanya data "yatim piatu" (orphan records) atau data rusak di dalam sistem.
