Bagian A — Persiapan & Query Sumber Data
Semua report dalam modul ini menggunakan Query sebagai sumber data, bukan tabel langsung
⚠️

Prasyarat: Modul ini melanjutkan database Modul 2 dan 4. Pastikan tabel customer, barang, dan transaksi sudah ada, terisi data, dan relasi antar tabel sudah dikonfigurasi dengan benar.

Mengapa Menggunakan Query sebagai Sumber?

Report langsung dari satu tabel saja tidak mampu menampilkan nama customer, nama barang, dan kalkulasi total dalam satu laporan. Query memungkinkan kita menggabungkan data dari beberapa tabel sekaligus sebelum dijadikan sumber report.

📌

Langkah wajib: Buat dua Query berikut di Query Design sebelum mengerjakan Tugas 1–6. Kedua query ini akan menjadi Record Source untuk seluruh report.

🔍 Query 1 — qry_TransaksiLengkap (digunakan di Tugas 1, 2, 3, 5)

Query ini menggabungkan ketiga tabel dan menambahkan kolom kalkulasi total_harga.

SQL — Query Design / SQL View
SELECT t.id AS id_transaksi, t.tgl_transaksi, c.nama AS nama_customer, c.kota, b.nama_barang, b.kategori, b.harga, t.jumlah, b.harga * t.jumlah AS total_harga FROM (transaksi t INNER JOIN customer c ON t.customer_id = c.id) INNER JOIN barang b ON t.barang_id = b.id ORDER BY t.tgl_transaksi, c.nama;
🔍 Query 2 — qry_RekapCustomer (digunakan di Tugas 4)

Query agregasi yang merangkum total belanja dan jumlah transaksi per customer.

SQL — Query Design / SQL View
SELECT c.nama AS nama_customer, c.kota, Count(t.id) AS jml_transaksi, Sum(b.harga*t.jumlah) AS total_belanja FROM (transaksi t INNER JOIN customer c ON t.customer_id = c.id) INNER JOIN barang b ON t.barang_id = b.id GROUP BY c.id, c.nama, c.kota ORDER BY Sum(b.harga*t.jumlah) DESC;
✅

Simpan kedua query dengan nama persis seperti di atas. Nama query akan direferensi langsung saat membuat report.

Report Wizard Grouping ● Mudah
Laporan Transaksi Dikelompokkan per Customer

Buat laporan yang menampilkan semua transaksi, dikelompokkan berdasarkan nama customer. Setiap kelompok menampilkan subtotal belanja dan di akhir laporan terdapat grand total keseluruhan.

1
Klik tab Create → Report Wizard. Pada dropdown Tables/Queries, pilih qry_TransaksiLengkap.
2
Pindahkan field berikut ke Selected Fields (urutan penting): nama_customer, kota, tgl_transaksi, nama_barang, jumlah, harga, total_harga. Klik Next.
3
Pada halaman Grouping Levels, klik nama_customer lalu klik tombol > untuk menjadikannya grup. Klik Next.
4
Pada Sort Order, pilih tgl_transaksi Ascending. Klik tombol Summary Options.
5
Di Summary Options, centang Sum untuk field total_harga. Pilih "Detail and Summary". Centang "Calculate percent of total for sums". Klik OK → Next.
6
Pilih layout Stepped, orientasi Portrait. Klik Next. Beri judul: "Laporan Transaksi per Customer". Pilih "Preview the report" → Finish.
7
Buka dalam Design View. Ubah nama report (Save As) menjadi rpt_TransaksiPerCustomer.

Design View Formatting ● Mudah
Memperindah Tampilan Report (Report Design)

Setelah report dibuat oleh Wizard, tampilan defaultnya masih terlihat sederhana. Gunakan Design View untuk menambahkan judul, warna, logo, dan format angka yang profesional.

1
Buka rpt_TransaksiPerCustomer dalam Design View. Perhatikan bagian-bagian report: Report Header, Page Header, Group Header, Detail, Group Footer, Page Footer, Report Footer.
2
Di Report Header: klik judul laporan, ubah ukuran font menjadi 18pt Bold. Tambahkan Text Box baru dengan Control Source: =Now() dan format: dd mmmm yyyy sebagai tanggal cetak.
3
Di Page Header: ubah background header kolom menjadi warna gelap (misal #2C3E50). Ubah warna teks header kolom menjadi putih. Atur font Bold.
4
Di Group Header (nama_customer Header): ubah background menjadi warna terang (misal #EBF5FB). Tambahkan garis bawah dengan Line control.
5
Pilih semua field angka (harga, total_harga, field Sum): atur Format = Currency di Property Sheet → tab Format.
6
Di Page Footer: tambah Text Box dengan Control Source: ="Halaman " & [Page] & " dari " & [Pages] di sisi kanan. Sisi kiri isi nama laporan/instansi.

Grouping Ganda Sorting ● Menengah
Laporan Penjualan Dikelompokkan per Kategori Barang

Buat laporan kedua yang mengelompokkan transaksi berdasarkan kategori barang (Elektronik, Aksesoris, Buku), lalu di dalam setiap kategori diurutkan berdasarkan nama barang dan total penjualan tertinggi.

1
Buat report baru via Create → Report Wizard, gunakan qry_TransaksiLengkap. Pilih field: kategori, nama_barang, nama_customer, tgl_transaksi, jumlah, total_harga.
2
Di Grouping Levels, tambahkan dua grup: pertama kategori, kedua nama_barang. Urutan: kategori → nama_barang.
3
Di Sort Order, pilih tgl_transaksi Ascending. Klik Summary Options → centang Sum untuk total_harga → pilih "Detail and Summary" → OK.
4
Pilih layout Outline, orientasi Landscape. Beri judul: "Laporan Penjualan per Kategori". Simpan sebagai rpt_PenjualanPerKategori.
5
Buka Design View. Di Group, Sort, and Total panel (klik Design → Group & Sort): pastikan kategori adalah grup pertama, nama_barang kedua. Aktifkan with a footer section untuk keduanya.
6
Di kategori Footer: tambahkan Text Box dengan Control Source =Sum([total_harga]) dan label "Total Kategori:". Format Currency.
7
Di nama_barang Footer: tambahkan Text Box =Sum([total_harga]) dengan label "Total Barang:" dan Text Box =Count([id_transaksi]) dengan label "Jml Transaksi:".

Agregasi Chart / Bar ● Menengah
Laporan Rekap Customer Terbaik

Buat laporan ringkasan yang menampilkan peringkat customer berdasarkan total belanja, dilengkapi persentase kontribusi dan visual bar sederhana.

1
Buat report baru via Create → Report Wizard, gunakan qry_RekapCustomer. Pilih semua field: nama_customer, kota, jml_transaksi, total_belanja.
2
Tidak perlu grouping di tahap ini. Untuk Sort: pilih total_belanja Descending (terbesar di atas). Klik Summary Options → Sum untuk total_belanja dan jml_transaksi.
3
Pilih layout Tabular, orientasi Portrait. Judul: "Rekap Belanja Customer". Simpan sebagai rpt_RekapCustomer.
4
Buka Design View. Di bagian Detail, tambah Text Box baru dengan nama txt_Ranking. Atur Control Source: =1 kemudian di Property Sheet → tab Data → Running Sum: Over All untuk nomor urut otomatis.
5
Tambah kolom persentase: Text Box dengan Control Source: =[total_belanja]/DSum("total_belanja","qry_RekapCustomer"). Format: Percent dengan 1 decimal.
6
Di Report Footer: tambah Text Box =Sum([total_belanja]) berlabel "Total Keseluruhan:" dan =Sum([jml_transaksi]) berlabel "Total Transaksi:".

5
Parameter Query Filter Tanggal ● Menengah
Laporan Transaksi dengan Filter Rentang Tanggal

Buat laporan yang menampilkan dialog saat dibuka, meminta pengguna memasukkan rentang tanggal. Hanya transaksi dalam rentang tersebut yang akan ditampilkan.

1
Buka qry_TransaksiLengkap dalam Design View. Di kolom tgl_transaksi, baris Criteria, ketik:
Between [Masukkan Tanggal Awal (dd/mm/yyyy):] And [Masukkan Tanggal Akhir (dd/mm/yyyy):]
2
Jangan simpan perubahan pada query ini (karena akan merusak report lain). Simpan dengan nama baru: Save As → qry_TransaksiByDate.
3
Buat report baru via Create → Report Wizard dari qry_TransaksiByDate. Pilih field: tgl_transaksi, nama_customer, nama_barang, jumlah, total_harga.
4
Group by: tgl_transaksi. Sort: nama_customer Ascending. Summary Options: Sum total_harga. Layout: Stepped, Portrait.
5
Simpan sebagai rpt_TransaksiByDate. Saat report dibuka di Print Preview, dialog tanggal akan muncul otomatis.
6
Di Report Header, tambahkan Text Box dengan Control Source: ="Laporan Transaksi: " & Format([Masukkan Tanggal Awal (dd/mm/yyyy):],"dd/mm/yyyy") & " s/d " & Format([Masukkan Tanggal Akhir (dd/mm/yyyy):],"dd/mm/yyyy") agar rentang tanggal tampil di judul laporan.
✅

Uji Coba: Buka report. Isi tanggal awal 01/01/2024 dan tanggal akhir 31/01/2024. Report harus menampilkan hanya 6 transaksi yang terjadi di bulan Januari 2024.

⚠️

Teks dalam kurung siku [ ] adalah nama parameter — harus diketik persis sama di semua tempat (criteria query dan Control Source di Report Header). Perbedaan satu huruf akan menyebabkan dialog muncul dua kali.

6
VBA DoCmd Lanjutan ● Sulit
Tombol Cetak & Ekspor Report dari Form

Tambahkan Command Button di form utama (frm_TransaksiCustomer dari Modul 4) yang memungkinkan pengguna mencetak atau mengekspor laporan ke PDF langsung dari form, tanpa harus membuka jendela report secara manual.

1
Buka frm_TransaksiCustomer dalam Design View. Tambah area baru atau gunakan Form Footer yang sudah ada.
2
Tambahkan 3 Command Button berdampingan: btn_PrintTransaksi, btn_PrintRekap, btn_ExportPDF. Beri label yang jelas di setiap tombol.
3
Untuk btn_PrintTransaksi, buka Property Sheet → Event → On Click → Code Builder. Isi kode VBA:
VBA — btn_PrintTransaksi On Click
Private Sub btn_PrintTransaksi_Click() ' Buka laporan transaksi per customer dalam Print Preview DoCmd.OpenReport "rpt_TransaksiPerCustomer", acViewPreview End Sub
4
Untuk btn_PrintRekap, isi event On Click:
VBA — btn_PrintRekap On Click
Private Sub btn_PrintRekap_Click() ' Cetak langsung ke printer default tanpa preview DoCmd.OpenReport "rpt_RekapCustomer", acViewNormal End Sub
5
Untuk btn_ExportPDF, isi event On Click dengan kode ekspor ke PDF:
VBA — btn_ExportPDF On Click
Private Sub btn_ExportPDF_Click() ' Ekspor rekap customer ke PDF di folder yang sama dengan database Dim strPath As String strPath = CurrentProject.Path & "\Rekap_Customer_" _ & Format(Now(), "yyyymmdd_HHmm") & ".pdf" DoCmd.OutputTo acOutputReport, _ "rpt_RekapCustomer", _ acFormatPDF, _ strPath, _ True ' True = buka file PDF setelah ekspor MsgBox "Laporan berhasil diekspor ke:" & vbCrLf & strPath, _ vbInformation, "Ekspor Selesai" End Sub
6
Simpan form dan uji ketiga tombol. Tombol ekspor PDF akan menyimpan file dengan nama otomatis berisi timestamp, sehingga tidak menimpa ekspor sebelumnya.
💡

Tentang acViewPreview vs acViewNormal: acViewPreview membuka Print Preview (pengguna bisa memeriksa sebelum cetak), sedangkan acViewNormal langsung mengirim ke printer default. Gunakan Preview untuk laporan penting agar pengguna bisa memverifikasi terlebih dahulu.
