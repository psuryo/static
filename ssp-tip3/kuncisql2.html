Kunci Jawaban — 15 Soal
MySQL / MariaDB syntax
1
Buku dengan status “rusak”Mudah
SQL
SELECT id, judul, pengarang, tahun
FROM   buku
WHERE  status = 'rusak';
Hasil: BUKU011 (Hujan) dan BUKU018 (Nebula), keduanya karya Tere Liye.
2
Customer dari Semarang atau BandungMudah
SQL
SELECT nama, kota
FROM   customer
WHERE  kota IN ('Semarang', 'Bandung')
ORDER BY nama ASC;
Alternatif: WHERE kota = 'Semarang' OR kota = 'Bandung'. Keduanya valid.
3
Buku karya Dee LestariMudah
SQL
SELECT judul, tahun, status
FROM   buku
WHERE  pengarang = 'Dee Lestari'
ORDER BY tahun;
Dee Lestari memiliki 14 judul dalam dataset (BUKU005, BUKU009, BUKU021–BUKU030). Cara LIKE '%Dee%' juga diterima.
4
Transaksi yang belum dikembalikanMudah
SQL
SELECT id, idcust, idbuku, tglpinjam
FROM   transaksi
WHERE  tglkembali IS NULL
ORDER BY tglpinjam;
Ada 7 transaksi dengan tglkembali NULL: TRX176, TRX177, TRX178, TRX181, TRX187, TRX191, TRX196.
5
Update status buku BUKU001Mudah
SQL
UPDATE buku
SET    status = 'tersedia'
WHERE  id = 'BUKU001';
Wajib ada WHERE! Tanpa WHERE akan mengubah status SEMUA buku menjadi 'tersedia'.
6
Riwayat peminjaman lengkap (3 tabel)Menengah
SQL
SELECT
  c.nama          AS nama_customer,
  b.judul         AS judul_buku,
  t.tglpinjam,
  t.tglkembali
FROM        transaksi t
INNER JOIN  customer  c ON t.idcust = c.id
INNER JOIN  buku      b ON t.idbuku = b.id
ORDER BY  t.tglpinjam DESC;
200 baris hasil. Urutan JOIN boleh dibalik selama klausa ON-nya konsisten.
7
Peminjaman dengan denda > Rp5.000Menengah
SQL
SELECT
  c.nama          AS nama_customer,
  b.judul         AS judul_buku,
  t.denda
FROM        transaksi t
INNER JOIN  customer  c ON t.idcust = c.id
INNER JOIN  buku      b ON t.idbuku = b.id
WHERE  t.denda > 5000
ORDER BY  t.denda DESC;
Denda tertinggi adalah Rp12.000 yang muncul di beberapa transaksi.
8
Frekuensi peminjaman per customerMenengah
SQL
SELECT
  c.nama              AS nama_customer,
  COUNT(t.id)         AS jumlah_pinjam
FROM        transaksi t
INNER JOIN  customer  c ON t.idcust = c.id
GROUP BY  c.id, c.nama
ORDER BY  jumlah_pinjam DESC;
CUST018 (Siti Aminah) adalah peminjam paling aktif dengan 18 transaksi.
9
Total denda per customer (hanya yang berdendam)Menengah
SQL
SELECT
  c.nama                AS nama_customer,
  SUM(t.denda)          AS total_denda
FROM        transaksi t
INNER JOIN  customer  c ON t.idcust = c.id
GROUP BY  c.id, c.nama
HAVING    SUM(t.denda) > 0
ORDER BY  total_denda DESC;
10
Top 10 buku terpopulerMenengah
SQL
SELECT
  b.judul                 AS judul_buku,
  b.pengarang,
  COUNT(t.id)             AS jumlah_dipinjam
FROM        transaksi  t
INNER JOIN  buku       b ON t.idbuku = b.id
GROUP BY  b.id, b.judul, b.pengarang
ORDER BY  jumlah_dipinjam DESC
LIMIT     10;
11
Buku yang tidak pernah dipinjamMenengah
SQL
SELECT
  b.id,
  b.judul,
  b.pengarang
FROM       buku       b
LEFT JOIN  transaksi  t ON b.id = t.idbuku
WHERE  t.id IS NULL;
Buku yang tidak pernah dipinjam antara lain: BUKU034 (Ancika 1995), BUKU044 (Rich Dad Poor Dad), BUKU047 (Filosofi Teras), dan beberapa lainnya.
12
Pengarang dengan total peminjaman ≥ 10Sulit
SQL
SELECT
  b.pengarang,
  COUNT(t.id)       AS total_dipinjam
FROM        transaksi  t
INNER JOIN  buku       b ON t.idbuku = b.id
GROUP BY  b.pengarang
HAVING    COUNT(t.id) >= 10
ORDER BY  total_dipinjam DESC;
Dee Lestari dan Tere Liye mendominasi daftar ini karena memiliki banyak judul dalam koleksi.
13
Peminjaman belum kembali > 14 hariSulit
SQL
SELECT
  c.nama              AS nama_customer,
  b.judul             AS judul_buku,
  t.tglpinjam,
  DATEDIFF(CURDATE(), t.tglpinjam) AS hari_terlambat
FROM        transaksi  t
INNER JOIN  customer   c ON t.idcust = c.id
INNER JOIN  buku       b ON t.idbuku = b.id
WHERE  t.tglkembali IS NULL
  AND  DATEDIFF(CURDATE(), t.tglpinjam) > 14
ORDER BY  hari_terlambat DESC;
Fungsi DATEDIFF dan CURDATE() adalah MySQL-specific. Untuk PostgreSQL gunakan: CURRENT_DATE - tglpinjam. Hasilnya bergantung pada tanggal eksekusi query.
14
Customer dengan total denda terbesarSulit
SQL
SELECT
  c.nama              AS nama_customer,
  c.kota,
  SUM(t.denda)        AS total_denda
FROM        transaksi  t
INNER JOIN  customer   c ON t.idcust = c.id
GROUP BY  c.id, c.nama, c.kota
ORDER BY  total_denda DESC
LIMIT     1;
CUST018 (Siti Aminah, Medan) adalah customer dengan total denda tertinggi.
15
Total peminjaman per kotaSulit
SQL
SELECT
  c.kota,
  COUNT(t.id)           AS total_peminjaman
FROM        transaksi  t
INNER JOIN  customer   c ON t.idcust = c.id
GROUP BY  c.kota
ORDER BY  total_peminjaman DESC;
Kota dengan aktivitas peminjaman tertinggi adalah Medan karena CUST018 (Siti Aminah) yang sangat aktif berasal dari sana.
