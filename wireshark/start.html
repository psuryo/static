01

Apa itu Wireshark? Instalasi & Capture Pertama

⏱ Estimasi waktu: 45 menit

Apa itu Wireshark?

Wireshark adalah penganalisis protokol jaringan (disebut juga packet sniffer) yang gratis dan open-source. Wireshark memungkinkan Anda melihat setiap paket data yang melewati antarmuka jaringan secara real time. Bayangkan seperti mesin X-ray untuk koneksi internet Anda — Anda bisa melihat persis data apa yang sedang dikirim dan diterima.


Penggunaan sah yang umum:

  • Pemecahan masalah jaringan (mengapa koneksi saya lambat?)
  • Analisis keamanan (mendeteksi lalu lintas mencurigakan)
  • Mempelajari cara kerja protokol jaringan
  • Mendeteksi transmisi data yang tidak aman (tidak terenkripsi)
  • Memverifikasi aturan firewall dan konfigurasi jaringan

Instalasi

Kunjungi https://www.wireshark.org/download.html dan unduh installer sesuai sistem operasi Anda (Windows, macOS, atau Linux).

Windows: Jalankan installer .exe. Saat diminta, install juga Npcap — ini diperlukan untuk live packet capture di Windows. Gunakan opsi default.

macOS: Buka file .dmg, seret Wireshark ke folder Applications. Anda mungkin perlu menginstall ChmodBPF untuk izin capture.

Linux (Ubuntu/Debian): Jalankan sudo apt install wireshark kemudian sudo usermod -aG wireshark $USER lalu restart komputer.

Buka Wireshark. Anda akan melihat daftar antarmuka jaringan dengan grafik lalu lintas langsung di sebelahnya.

Antarmuka Wireshark — Orientasi Awal

Saat membuka Wireshark, Anda akan melihat tiga panel utama:


PanelYang Ditampilkan
Packet List (atas)Satu baris per paket — waktu, IP sumber, IP tujuan, protokol, info
Packet Details (tengah)Pohon yang bisa diperluas berisi semua lapisan protokol dari paket yang dipilih
Packet Bytes (bawah)Representasi hex dan ASCII mentah dari data paket

Capture Pertama Anda — Langkah demi Langkah

Buka Wireshark. Di layar awal, Anda akan melihat daftar antarmuka jaringan. Cari Wi-Fi atau Ethernet — yang memiliki grafik gelombang hidup di sebelahnya berarti sedang aktif.

Klik dua kali antarmuka aktif Anda (misalnya Wi-Fi). Wireshark langsung mulai menangkap semua lalu lintas.

Buka browser Anda dan kunjungi http://example.com (perhatikan: HTTP, bukan HTTPS).

Kembali ke Wireshark. Anda akan melihat puluhan atau ratusan paket bergulir. Tekan tombol Stop kotak merah (■) di toolbar untuk menghentikan capture.

Di bilah filter di bagian atas, ketik http dan tekan Enter. Anda sekarang hanya akan melihat paket HTTP — termasuk permintaan yang dibuat browser Anda ke example.com.

Klik salah satu paket HTTP tersebut. Di panel tengah, perluas bagian Hypertext Transfer Protocol untuk membaca header HTTP lengkapnya.

💡
Tips: Simpan file capture Anda dengan File → Save As menggunakan format .pcap atau .pcapng. Anda bisa membukanya kapan saja — sangat berguna untuk latihan tanpa perlu melakukan capture ulang.
02

Memahami Paket & Model OSI

⏱ Estimasi waktu: 50 menit

Apa itu Paket?

Data jaringan dipecah menjadi potongan-potongan kecil yang disebut paket. Setiap paket berisi:

  • Header — metadata (sumber, tujuan, jenis protokol, nomor urut)
  • Payload — data aktual yang sedang dikirimkan

Saat Anda mengunjungi sebuah website, browser Anda tidak mengirim satu permintaan besar — melainkan memecahnya menjadi banyak paket. Wireshark menampilkan semuanya, satu per satu.

Model OSI — Mengapa Penting di Wireshark

Wireshark mengorganisasi informasi paket sesuai dengan model OSI 7 lapisan. Saat Anda mengklik paket dan memperluasnya di panel detail, Anda akan melihat setiap lapisan:


LapisanNamaYang Ditampilkan WiresharkContoh Protokol
7AplikasiPermintaan/respons HTTP, query DNSHTTP, DNS, FTP
6PresentasiEnkripsi/encoding (record TLS)TLS/SSL
5SesiPembentukan sesiNetBIOS
4TransportNomor port, nomor urutTCP, UDP
3JaringanAlamat IP, routingIP, ICMP
2Data LinkAlamat MACEthernet, ARP
1Fisik(Tidak terlihat di Wireshark — level hardware)—

Buka capture yang sudah Anda simpan (atau lakukan yang baru). Temukan paket HTTP dan klik. Di panel tengah, Anda akan melihat seperti ini:

Atur capture filter sebelum memulai dengan memasukkannya di bilah capture filter pada layar awal.

Latihan Praktik

Mulai capture, buka browser dan muat beberapa website. Hentikan capture. Terapkan filter dns — perhatikan bahwa setiap website yang Anda kunjungi memicu DNS lookup terlebih dahulu.

Hapus filter. Terapkan http.request.method == "GET" untuk melihat semua permintaan GET. Perhatikan berapa banyak permintaan yang dihasilkan oleh satu halaman (gambar, CSS, file JS).

Terapkan filter gabungan: http && ip.dst != 192.168.1.1 untuk melihat lalu lintas HTTP yang melewati router Anda.

Klik kanan paket mana pun → Apply as Filter → Selected untuk membuat filter cepat berdasarkan IP atau protokol tersebut.

04

Intersepsi Lalu Lintas HTTP pada Website Tidak Aman

⏱ Estimasi waktu: 60 menit
🔒
Pengingat etika: Sesi ini mendemonstrasikan mengapa HTTP (bukan HTTPS) sangat berbahaya. Semua contoh di bawah harus dipraktikkan pada server uji milik sendiri atau pada lingkungan pelatihan yang sengaja dibuat rentan seperti DVWA (Damn Vulnerable Web Application) yang berjalan di localhost. Jangan pernah melakukan ini pada website yang bukan milik Anda.

HTTP vs HTTPS — Perbedaan Utama

HTTP (Port 80)HTTPS (Port 443)
Enkripsi❌ Tidak ada — teks biasa✅ Enkripsi TLS
Wireshark bisa membaca?✅ Ya, sepenuhnya terbaca❌ Hanya ciphertext terenkripsi
Form login aman?❌ Kredensial terlihat jelas✅ Kredensial terenkripsi
Tampilan URL barPeringatan "Tidak aman"🔒 Ikon gembok

Contoh 1: Melihat Permintaan HTTP GET

Saat browser Anda memuat halaman melalui HTTP, seluruh permintaan terlihat dengan jelas:

// Paket Wireshark — HTTP GET (Teks Biasa)
GET /index.html HTTP/1.1
Host: testphp.vulnweb.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Accept: text/html,application/xhtml+xml
Accept-Language: id-ID,id;q=0.9
Connection: keep-alive

Anda bisa melihat halaman persis yang diminta, jenis browser, dan sistem operasi yang digunakan.

Contoh 2: Mengintersep Form Login melalui HTTP

Ini adalah demonstrasi paling kritis mengapa HTTP sangat berbahaya. Ketika form login dikirim melalui HTTP (tanpa TLS), username dan password berjalan dalam teks biasa:

Siapkan uji lokal. Install DVWA di mesin Anda (via XAMPP/WAMP) atau gunakan halaman login lokal yang Anda buat sendiri. Pastikan menggunakan http://localhost bukan https.

Mulai capture Wireshark pada antarmuka Loopback (lo) (untuk lalu lintas localhost) atau antarmuka Wi-Fi Anda.

Di DVWA, navigasi ke halaman login dan masukkan username percobaan (misalnya admin) dan password (misalnya password123). Klik Login.

Hentikan capture di Wireshark. Di bilah filter, ketik: http.request.method == "POST"

Klik paket POST tersebut. Di panel detail paket, perluas Hypertext Transfer Protocol → HTML Form URL Encoded.

// Yang Diungkap Wireshark — HTTP POST Login
POST /login.php HTTP/1.1
Host: localhost
Content-Type: application/x-www-form-urlencoded
Content-Length: 38

▼ HTML Form URL Encoded:
  username = admin
  password = password123
  Login    = Login
⚠️
Username dan password terlihat sepenuhnya. Siapa pun di jaringan yang sama yang menjalankan Wireshark bisa melihat ini. Inilah tepatnya mengapa setiap website yang menangani login WAJIB menggunakan HTTPS.

Contoh 3: Mengikuti Seluruh Aliran HTTP

Wireshark dapat merekonstruksi seluruh percakapan HTTP (permintaan + respons) dalam satu tampilan yang mudah dibaca:

Klik kanan pada paket HTTP mana pun di capture Anda.

Pilih Follow → HTTP Stream.

Jendela baru terbuka menampilkan percakapan lengkap — teks merah adalah permintaan Anda, teks biru adalah respons server.

// Follow HTTP Stream — Pertukaran Lengkap
→ Permintaan Client (merah):
POST /login.php HTTP/1.1
Host: testsite.local
Content-Type: application/x-www-form-urlencoded

user=admin&pass=rahasia99&submit=Login

← Respons Server (biru):
HTTP/1.1 302 Found
Location: /dashboard.php
Set-Cookie: PHPSESSID=abc123xyz; path=/

Perhatikan respons server menyertakan cookie sesi. Siapa pun yang menangkap cookie ini berpotensi menggunakannya untuk menyamar sebagai pengguna yang sudah login — teknik ini disebut session hijacking.

Contoh 4: Mengekstrak Cookie dari Lalu Lintas HTTP

Cookie yang dikirim melalui HTTP juga dalam teks biasa. Gunakan filter ini untuk menemukannya:

// Display Filter
http.cookie

Kemudian klik hasil mana pun dan lihat di dalam header HTTP untuk field Cookie:. Melalui HTTP, ini mencakup token sesi, preferensi pengguna, dan terkadang nilai autentikasi — semuanya terbaca.

Contoh 5: Tampilan HTTPS sebagai Perbandingan

Sebagai perbandingan, terapkan filter tcp.port == 443 pada website HTTPS mana pun. Klik paket dan lihat payload-nya:

// Payload Paket HTTPS (Terenkripsi)
▼ Transport Layer Security
  TLS Record Layer: Application Data Protocol
  Content Type: Application Data (23)
  Version: TLS 1.3
  Encrypted Application Data:
  17 03 03 00 45 a3 f2 8c 1e 4d 22 b9 ...
  (ciphertext yang sama sekali tidak terbaca)

Tidak ada konten yang terbaca. Payload sepenuhnya terenkripsi. Inilah mengapa ikon gembok di browser Anda sangat penting.

05

Analisis Protokol: DNS, ARP, TCP, FTP

⏱ Estimasi waktu: 55 menit

Analisis DNS — Sedang Berbicara dengan Siapa?

DNS (Domain Name System) mengubah nama domain menjadi alamat IP. Setiap kunjungan website dimulai dengan query DNS — dan dikirim dalam teks biasa, bahkan di situs HTTPS sekalipun.

// Filter: dns
Query DNS (PC Anda → server DNS):
  Name: www.example.com
  Type: A (Host Address)

Respons DNS (server DNS → PC Anda):
  Name: www.example.com
  Address: 93.184.216.34

Yang terungkap: Setiap domain yang pernah dicari oleh perangkat Anda — bahkan jika Anda tidak pernah benar-benar memuat halamannya. Lalu lintas DNS adalah peta aktivitas browsing.


Coba filter ini untuk menemukan semua domain yang pernah di-query:

// Display Filter
dns.flags.response == 0   ← Tampilkan hanya query (bukan respons)

Analisis ARP — Memetakan Jaringan Lokal

ARP (Address Resolution Protocol) memetakan alamat IP ke alamat MAC pada jaringan lokal Anda. Beroperasi di Layer 2 dan digunakan setiap kali perangkat Anda berkomunikasi dengan perangkat lain di jaringan yang sama.

// Filter: arp
Permintaan ARP (broadcast ke semua perangkat):
  Siapa yang punya 192.168.1.1? Beritahu 192.168.1.100

Balasan ARP:
  192.168.1.1 ada di A4:C3:F0:85:90:01

Catatan keamanan: ARP spoofing/poisoning melibatkan pengiriman balasan ARP palsu untuk mengalihkan lalu lintas melalui perangkat penyerang — vektor serangan man-in-the-middle yang umum. Melihat balasan ARP berlebihan dari alamat MAC yang tidak dikenal bisa menjadi tanda serangan ini.

Analisis Handshake TCP

Koneksi TCP selalu dimulai dengan 3-way handshake. Memahaminya membantu Anda mengidentifikasi masalah koneksi.

// Filter: tcp.flags.syn == 1
Langkah 1 — SYN (Client → Server):
  Flags: SYN
  Seq: 0

Langkah 2 — SYN-ACK (Server → Client):
  Flags: SYN, ACK
  Seq: 0, Ack: 1

Langkah 3 — ACK (Client → Server):
  Flags: ACK
  Seq: 1, Ack: 1

→ Koneksi terjalin. Transfer data dimulai.
Filter TCPTujuan
tcp.flags.syn == 1Inisiasi koneksi (paket SYN)
tcp.flags.fin == 1Penutupan koneksi (paket FIN)
tcp.flags.reset == 1Reset koneksi mendadak (RST) — sering menandakan koneksi yang ditolak
tcp.analysis.retransmissionPaket yang dikirim ulang — tanda kehilangan paket
tcp.analysis.zero_windowBuffer penerima penuh — menyebabkan perlambatan

Analisis FTP — Kredensial dalam Teks Biasa

FTP (File Transfer Protocol) adalah protokol lain yang mengirimkan kredensial login dan data file dalam teks biasa. Contoh klasik protokol tidak aman yang masih ditemukan di sistem lama.

// Filter: ftp
Client → Server:
  USER ftpadmin
Server → Client:
  331 Password diperlukan untuk ftpadmin.
Client → Server:
  PASS passwordrahasia
Server → Client:
  230 Pengguna ftpadmin berhasil login.

Username dan password terlihat sepenuhnya. Untuk melihat file yang ditransfer, gunakan:

// Filter untuk Transfer Data FTP
ftp-data

Kemudian klik kanan paket data → Follow → TCP Stream untuk melihat isi file yang sedang ditransfer.

06

Skenario Dunia Nyata & Lab Praktik

⏱ Estimasi waktu: 40 menit

Skenario 1: Mendiagnosis Halaman yang Lambat Dimuat

Sebuah halaman web lambat dimuat. Gunakan Wireshark untuk mengetahui penyebabnya.

Mulai capture, muat halaman yang lambat, hentikan capture.

Pergi ke Statistics → Service Response Time → HTTP untuk melihat permintaan mana yang paling lama.

Terapkan tcp.analysis.retransmission — retransmisi berarti kehilangan paket dan masalah jaringan.

Gunakan Statistics → IO Graph untuk memvisualisasikan lalu lintas dari waktu ke waktu dan menemukan lonjakan atau penurunan.

Skenario 2: Mengidentifikasi Lalu Lintas Tidak Dikenal di Jaringan

Capture selama 5 menit tanpa aktivitas. Terapkan ip.addr != ip_anda atau cari lalu lintas dari IP yang tidak dikenal.

Gunakan Statistics → Endpoints untuk melihat daftar semua IP yang berkomunikasi. Urutkan berdasarkan paket yang dikirim.

Gunakan Statistics → Conversations untuk melihat dua perangkat mana yang paling banyak berkomunikasi. Selidiki alamat yang tidak dikenal dengan pencarian WHOIS.

Filter dns untuk melihat domain apa yang di-query oleh perangkat yang tidak dikenal.

Skenario 3: Mendeteksi Login HTTP pada Halaman yang Dikompromikan

Bayangkan halaman login palsu (halaman phishing) yang berjalan melalui HTTP. Begini tampilan serangan di Wireshark:

// Permintaan POST ke halaman phishing
POST /secure-login.php HTTP/1.1
Host: bank-secure-login.example.com   ← Domain palsu!
Content-Type: application/x-www-form-urlencoded

email=korban%40gmail.com&password=PasswordBank99

Di Wireshark, Anda akan melihat: domain terlihat sah sekilas namun bukan domain bank yang sebenarnya, koneksi adalah HTTP (tanpa TLS), dan kredensial terlihat jelas. Inilah mengapa memeriksa bilah URL dan ikon gembok sangat penting sebelum mengetik password apapun.

Skenario 4: Memantau Percobaan DNS Poisoning

Dalam serangan DNS spoofing, pelaku mengirim balasan DNS palsu sebelum yang asli tiba, mengalihkan pengguna ke server palsu. Filter untuk menemukan anomali:

// Deteksi respons DNS duplikat
dns.flags.response == 1 && dns.count.answers > 0

Jika Anda melihat dua respons DNS untuk query yang sama dengan alamat IP berbeda, itu adalah tanda bahaya poisoning. Respons penyerang biasanya tiba lebih dulu dengan IP internal atau mencurigakan.
